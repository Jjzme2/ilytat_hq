## 2026-03-01 - [Critical] Unsafe Try-Catch around Admin Auth
**Vulnerability:** Several admin API endpoints (`users.get.ts`, `invite.post.ts`, etc.) wrapped `verifyAdminAccess(event)` in a `try-catch` block that suppressed errors, allowing unauthorized access to sensitive functionality.
**Learning:** The "Fail Open" anti-pattern (swallowing errors in security checks) is extremely dangerous. It likely originated from a desire to support local development without credentials, but it compromised production security.
**Prevention:** Security checks must always "Fail Closed". Never wrap `verifyAdminAccess` or similar authorization logic in a `try-catch` block that doesn't re-throw or explicitly handle the denial. If local dev support is needed, it must be explicit and safe (e.g., via a mock service, not by bypassing the check in the production code path).
