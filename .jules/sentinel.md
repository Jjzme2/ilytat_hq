## 2024-05-22 - [Missing Authentication in Document API]
**Vulnerability:** The `server/api/docs` endpoints (GET, POST, DELETE) lacked authentication checks, allowing unrestricted access to sensitive document storage (R2 bucket).
**Learning:** Nuxt server routes do not inherit authentication automatically. `server/utils/adminAuth.ts` exists but was not utilized in these endpoints.
**Prevention:** Always invoke `verifyAdminToken(event)` at the beginning of `defineEventHandler` for any API route that handles sensitive data or operations.
## 2026-02-13 - [Critical] Unprotected S3 Proxy Endpoint
**Vulnerability:** The `server/api/docs.get.ts` endpoint proxied requests to Cloudflare R2 without any authentication or authorization checks.
**Learning:** Server-side API routes that act as proxies to external storage must always validate authentication before forwarding requests. Relying on obfuscated URLs is not security.
**Prevention:** Always add `verifyAdminToken` or equivalent middleware at the start of any server-side API handler that accesses sensitive data.

## 2026-02-13 - [Critical] Admin Authentication Bypass / Fail-Open
**Vulnerability:** Many endpoints in `server/api/admin/` were wrapping their `verifyAdminAccess` call inside a `try/catch` block with an empty `catch` body (commented `// Local dev fallback`), or returning a hardcoded positive response in `catch`. This caused the authentication check to fail open. An attacker calling the admin API without credentials (or bad credentials) would silently have their lack of authentication ignored, continuing to process the endpoint logic with potentially unauthenticated contexts or unassigned variables.
**Learning:** Suppressing authentication errors (Fail Open) is a strictly forbidden security anti-pattern. Admin API routes (`server/api/admin/*`) must strictly await `verifyAdminAccess(event)` at the handler's start without wrapping it in a `try-catch` block, allowing unauthorized access attempts to throw the correct `createError` generated by `verifyAdminAccess`.
**Prevention:** Never use a `try/catch` block to suppress authentication logic exceptions for "local dev fallbacks". Use proper mocked authentication or an explicit environment switch (`process.dev`) if absolutely necessary, but strictly enforce the `verifyAdminAccess` throwing behavior in code.
