## 2024-03-XX - [N+1 Transaction Anti-Pattern in Users Listing]
**Learning:** In `server/api/admin/users.get.ts`, auto-assigning incrementing `employeeId` during map enumeration triggers individual transactions per user missing an `employeeId`. This N+1 transaction pattern can cause contention and significant delays if multiple users in the same tenant require an ID simultaneously.
**Action:** When assigning incrementing IDs to multiple documents, avoid N+1 transaction patterns and counter document contention by grouping updates by their parent scope (e.g., `tenantId`) and performing a single transaction per group.
